# Question Generation Module

This module makes quiz questions from a student's study material and checks their
answers. It's a FastAPI service. The questions are generated by an LLM, but the
difficulty is not left to the LLM's word — we validate it with a fine-tuned
BloomBERT classifier so an "easy" question is actually easy.

Two things make the output better than just prompting a model:

- **ICL (in-context learning)**: before generating, we retrieve a few expert
  example questions at the requested level from an example bank and put them in
  the prompt. The model copies the style and difficulty instead of guessing it.
- **BloomBERT difficulty check**: a DistilBERT model trained on Bloom-taxonomy
  labels predicts the level of each generated question. If it doesn't match what
  was asked, we can retry.

## Layout

```
app/
  main.py            FastAPI app + startup wiring
  config.py          settings
  routes/
    question_routes.py   /questions/generate, /questions/check
    adaptive_routes.py   adaptive session endpoints
  services/
    question_service.py   the generation flow (retrieve examples -> prompt -> validate)
    example_bank.py       loads + retrieves few-shot examples
    chunk_fetcher.py      pulls chunks from the RAG/parsing side
    answer_grader.py      grades a student's answer
    semantic_validator.py checks the question is on-topic
    context_enrichment.py extra context for the prompt
    adaptive_session.py   tracks a student's session
  classifier/
    bloom_classifier.py   BloomBERT wrapper (load + predict)

scripts/        offline data + evaluation scripts (see below)
notebooks/      BloomBERT training notebooks
models/         the trained bloom_distilbert model
data/           raw + processed datasets, example bank
```

The LLM clients and prompts are kept separate from the logic on purpose. The
generation prompts live in `../question_generation_model/prompt_builder.py`, the two API clients
in `../question_generation_model/llm/`, and in the evaluation scripts the prompt + client wiring sit in
`../question_generation_model/_judge_llm.py` and `../question_generation_model/_gen_llm.py`. So the scripts themselves stay
about the experiment, not the API plumbing.

## Running it

```bash
pip install -r requirements.txt
cp .env.example .env    # add GROQ_API_KEY and/or GEMINI_API_KEY
uvicorn app.main:app --port 8001
```

The example bank and the BloomBERT model need to exist before generation works.
The model is under `models/bloom_distilbert/`. If the example bank is missing,
build it:

```bash
python scripts/build_example_bank.py
```

## Scripts

These are offline, not part of the API.

- `build_example_bank.py` — builds the few-shot example bank + embeddings from
  the labeled CSVs and the curated OS questions
- `build_training_set.py` — assembles the BloomBERT training data
- `parse_os.py`, `parse_srm.py`, `relabel_os.py` — dataset prep
- `generate_baseline_vs_icl.py` — generates questions twice per cell (with and
  without the example bank) for the comparison
- `llm_judge_baseline_vs_icl.py` — blinded pairwise LLM-as-judge over those
  generations, with counterbalanced ordering to cancel position bias
- `difficulty_match_baseline_vs_icl.py` — the judge-free check: runs BloomBERT on
  every generated question and reports how often the predicted level matches the
  requested one

## Notebooks

`notebooks/` has the BloomBERT training runs. Each variant tries a different
training setup (focal loss, cosine schedule, label smoothing, mean pooling, and
combinations) to push the difficulty classifier's accuracy. The trained model
that the app actually loads is the one saved under `models/bloom_distilbert/`.
